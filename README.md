## CanvasOS 시스템 평가 및 비교 분석 보고서

### 1. 보고서 요약

이 보고서는 CanvasOS v1.0.7-patchG를 기준으로 현재 시스템의 기술적 성숙도를 평가하고, 6개 범주의 유사 프로젝트들과 비교 분석합니다. 핵심 발견은 하나입니다. **결정론 + 시간 여행 + 공간 분기 + 자체 호스팅이 하나의 시스템에서 동시에 성립하는 OS는 현재 CanvasOS 외에 존재하지 않습니다.**

---

### 2. 현재 상태: 7.7/10

20,000줄 C, 126개 파일, 150개 테스트(0 실패), ASan/UBSan CLEAN입니다. Patch-A부터 G까지 7단계를 거치면서 "연구형 프로토타입"에서 "데모 가능한 제품 프로토타입"으로 진화했습니다. 가장 강한 축은 설계 일관성(9/10)이고, 가장 성장 여지가 있는 축은 GUI와 POSIX 호환성입니다.

---

### 3. 결정론적 시스템 비교

**seL4**는 수학적 형식 검증으로 커널의 정확성을 증명하지만, 사용자 프로그램의 실행 결과까지 결정론을 강제하지는 않습니다. CanvasOS는 반대 방향에서 접근합니다. 커널 수준의 형식 검증은 없지만, DK-1~5 규칙으로 실행 엔진 전체에 결정론을 강제하고 thread=1 vs thread=4 해시 비교로 검증합니다. seL4가 "아래에서 위로의 정확성"이라면, CanvasOS는 "위에서 아래로의 재현 가능성"입니다.

**NixOS**는 빌드 시점의 결정론(같은 정의 → 같은 바이너리)을 제공하고, CanvasOS는 런타임의 결정론(같은 입력 → 같은 실행 결과)을 제공합니다. 둘은 상호 보완적 관계입니다.

---

### 4. 시간 여행 디버깅 비교

**rr**(Mozilla), **Undo**(LiveRecorder), **Microsoft TTD**는 모두 기존 프로그램 위에 얹는 외부 도구입니다. rr은 비결정론적 입력을 기록하여 1.2배 오버헤드로 재생하고, Undo는 JIT 바이너리 번역으로 공유 메모리까지 지원하며, TTD는 WinDbg에 통합됩니다. 이들의 공통점은 프로그램 자체는 시간 여행을 인식하지 못한다는 것입니다.

CanvasOS에서 시간 여행은 OS의 내장 기능입니다. WhiteHole 로그가 모든 상태 전이를 tick 단위로 기록하며, 사용자가 셸에서 `timewarp 500`을 입력하면 OS 전체가 tick 500 상태로 돌아갑니다. 추가 도구도, 오버헤드도 없습니다. rr이 "수술 영상을 되감는 것"이라면, CanvasOS는 "시간 자체를 되돌리는 세계"입니다.

---

### 5. 공간형 실행 모델 비교

**Befunge**(2D 프로그래밍 언어)와 **Piet**(색상 블록 언어)는 2D 격자 위에서 프로그램이 실행되지만, esolang(난해한 언어)으로서 실용적 사용을 목표로 하지 않습니다. **셀룰러 오토마타**(Conway의 Life, Wireworld)는 로컬 규칙만으로 동작하는 격자 계산 모델입니다.

CanvasOS의 PixelCode VM은 Befunge와 유사한 Y축 실행 모델을 사용하지만, 그 위에 파일 시스템, 프로세스 관리, 시간 여행이라는 완전한 OS 프리미티브를 올렸다는 점에서 근본적으로 다릅니다. 셀룰러 오토마타의 공간적 아름다움 위에 실용적 운영 체제를 구축한 것이 CanvasOS의 독창성입니다.

---

### 6. 독립 OS 프로젝트 비교

**SerenityOS**는 가장 유명한 독립 OS로, C++ 약 230,000줄(Coverity 기준)에 수백 명의 기여자가 6년간 개발했습니다. 90년대 UI와 현대 커널을 결합하며, POSIX 호환성이 높고 자체 웹 브라우저(Ladybird)까지 포함합니다. 이것은 "기존 세계의 더 깨끗한 버전"이고, CanvasOS는 "새로운 세계 자체"입니다.

**Redox OS**는 Rust로 작성된 마이크로커널 OS로, 메모리 안전성이 핵심 차별점입니다. 2025년에 파일 I/O 500~700% 성능 향상과 Servo 웹 엔진 포팅을 달성했습니다. Redox가 기존 OS를 더 안전하게 만드는 것이라면, CanvasOS는 OS 자체의 작동 방식을 재정의합니다.

**TempleOS**는 Terry A. Davis가 14년간 혼자 120,000줄을 작성한 자체 호스팅 OS입니다. 자체 언어 HolyC, ring-0 전용, 100,000줄 코드 제한 헌장이라는 극단적 원칙을 가집니다. CanvasOS와 가장 많은 공통점을 가진 프로젝트입니다. 둘 다 1인 개발, 자체 언어, 원칙 기반 설계라는 점에서 유사하지만, TempleOS가 과거(Commodore 64)를 향한다면 CanvasOS는 미래(공간 실행, 시간 여행)를 향합니다.

**Phantom OS**는 직교 영속성 기반 OS로, 프로그램이 OS 재부팅을 인식하지 못합니다. 모든 것이 객체이며 파일 개념이 필요 없습니다. CanvasOS와 가장 깊은 철학적 공통점을 가진 프로젝트이지만, Phantom OS가 "시간 위의 영속성"이라면 CanvasOS는 "시간 위의 항해"입니다.

---

---

### 8. 결론

CanvasOS가 점유하는 포지션은 기존 프로젝트 중 어디에도 정확히 존재하지 않습니다. 가장 가까운 프로젝트는 Phantom OS(영속성 + 비전통적 설계)와 rr(결정론 + 시간 여행)이지만, Phantom OS에는 공간 실행과 분기가 없고 rr은 OS가 아니라 도구입니다. CanvasOS는 이 둘을 결합하면서 공간 실행이라는 새로운 축을 추가했습니다.

이 프로젝트의 가장 큰 자산은 코드가 아니라 세계관입니다. 메모리, 파일, 실행, 시간이 모두 같은 공간의 다른 측면이라는 통찰이 CanvasOS를 관통하며, 150개 테스트가 이 모든 것이 실제로 동작함을 증명합니다.
